# Ralph Progress Log

## Codebase Patterns
- `app/rules/reducer.py`: pure function `apply_intent(state, intent, card_lookup, rng)` ‚Äî all game logic here; no side effects
- `app/rules/captures.py`: `resolve_captures(board, placed_index, placed_card, placed_owner, card_lookup, mists_modifier, presence_direction)` ‚Äî BFS queue; combo entries get zero modifier
- `app/rules/cards.py`: `validate_card_balance(card)` returns list[str] of errors; `rarity_bucket(rarity)` maps int‚Üístr
- `app/models/game.py`: `LastMoveInfo` and `GameState` Pydantic models; additive fields are safe (JSONB snapshot)
- `app/models/cards.py`: `CardDefinition` Pydantic model with `CardSides`
- `cards.jsonl`: JSON Lines, one card per line; 65 cards total; loaded at startup via `app/rules/cards.py`
- Frontend tests use vitest + jsdom; mock API via `vi.mock('../../lib/api')`; use `mock`-prefixed vars in vi.mock factories
- `bun run test --run` for frontend; `uv run pytest` for backend
- Commit format: `feat: [US-XXX] - Title`

## Context: Why These Mechanics

Analysis run before this sprint revealed:
- Ultra cards average 8.0/side vs common cards at 4.0/side ‚Äî a 4-point gap
- Ultra‚ÜíCommon win rate per comparison: 89%; Common‚ÜíUltra: 3.3%
- Mists ¬±1 only flips a capture outcome in ~8.5% of placements ‚Äî effectively noise
- Plus rule (simulated): would trigger on 22% of eligible placements ‚Äî meaningful without overwhelming
- These four mechanics address the core "race to play high-power cards" problem
